# Браузерный анализатор аккордов guitar.ai

Проект для анализа аудио-файлов с распознаванием гитарных аккордов и генерацией табулатур.

## Требования к системе

Для локальной разработки вам понадобятся:

- **Node.js** (версия 18 или выше) - [скачать Node.js](https://nodejs.org/)
- **npm** (обычно устанавливается вместе с Node.js)

Проверьте установку:

```bash
node --version
npm --version
```

## Установка зависимостей

1. Перейдите в директорию проекта:

```bash
cd C:\work\vcs\ailjushkin\projects\chords_analyzer
```

2. Установите все необходимые зависимости:

```bash
npm install
```

Это установит все пакеты, указанные в `package.json`.

## Запуск проекта для локальной отладки

Для удобства разработки созданы скрипты запуска:

### Windows (bat файлы)

```bash
start-dev.bat
```

или

```bash
start-prod.bat
```

### Linux/macOS (sh файлы)

```bash
./start-dev.sh
```

или

```bash
./start-prod.sh
```

После запуска приложение будет доступно по адресу `http://localhost:3000`

```

│   ├── utils/           # Утилиты
│   │   ├── helpers.js

│   ├── App.js           # Основной компонент приложения
│   ├── App.css          # Стили приложения
│   ├── index.js         # Точка входа
```

## Особенности реализации

- Код модульный и логически разделяется по функциям
- Подробное логирование в консоль для легкого дебага
- Асинхронная обработка аудио файлов для предотвращения блокировки UI
- Используются готовые фреймворки и библиотеки

## Скрипты npm

- `npm start` - Запуск сервера разработки на `http://localhost:3000`
- `npm run build` - Создание продакшн сборки в папке `build`
- `npm test` - Запуск тестов в интерактивном режиме
- `npm run eject` - "Извлечение" конфигурации (одноразовое действие)
- `npm run lint` - Проверка кода с помощью ESLint
- `npm run format` - Форматирование кода с помощью Prettier

## Функционал

- Загрузка аудио-треков
- Выбор типа гитары (по умолчанию 6-струнная)
- Транспонирование табулатуры
- Визуализация осциллограммы
- Воспроизведение аудио с тайм-кодами
- Отображение распознанных аккордов с табулатурами
- История проанализированных файлов

## Архитектура

### Services (Сервисы)
Модули для работы с аудио и внешними API:
- `audioAnalyzer.js` - Анализ аудио и извлечение аккордов

### Utils (Утилиты)
Вспомогательные функции:
- `helpers.js` - Утилиты для работы со временем, аккордами и аудио
- `constants.js` - Константы проекта (типы гитар, этапы анализа и т.д.)

### Компоненты
Корневой компонент `App.js` управляет всем приложением и содержит:
- Логику загрузки файлов
- Управление состоянием аудио
- Работу с аудио контекстом
- Отображение интерфейса
